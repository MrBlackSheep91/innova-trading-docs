## EA Connection Errors {#ea-errors}

These errors appear in your MetaTrader **Experts** tab when the InnovaTrading EA has issues.

### Error 401 - Unauthorized

```
InnovaTrading EA: HTTP Error 401 - Unauthorized
```

**Causes:**
- Invalid API token
- Token has been revoked
- Token has expired (after 1 year)

**Solution:**
1. Go to [Connections](https://innova-trading.com/dashboard/connect)
2. Generate a new token
3. Update the `API_TOKEN` in your EA settings

---

### Error 4014 / Error 4060 - WebRequest Failed

```
InnovaTrading EA: Error 4014 - WebRequest to URL failed
```
or
```
InnovaTrading EA: Error 4060 - Function is not allowed for call
```

**Cause:** MetaTrader doesn't allow WebRequest to our server URL.

**Solution:**
1. Go to **Tools** → **Options** → **Expert Advisors**
2. Check ✅ **Allow WebRequest for listed URL**
3. Click **Add** and enter: `https://cloud.innova-trading.com`
4. Click **OK** and restart the EA

### EA Shows "Waiting..." Forever

```
InnovaTrading EA: Waiting for connection...
```

**Causes:**
1. Token field is empty
2. Token is invalid
3. Network connectivity issue

**Solution:**
1. Double-click the EA on your chart to open settings
2. Verify `API_TOKEN` field has your token (not empty)
3. Check your internet connection
4. Try disabling firewall temporarily

---

### Smiley Face Shows X (Auto-Trading Disabled)

**Cause:** MetaTrader's global auto-trading is disabled.

**Solution:**
1. Click the **AutoTrading** button in MT4/MT5 toolbar (should turn green)
2. Or: Right-click the EA → **Allow Algorithmic Trading**

---

### No Data Appears on Dashboard

**Possible causes:**
1. EA just started - wait 10-15 seconds
2. Token not linked to your account yet
3. Backend temporarily unavailable

**Solution:**
1. Wait 15 seconds and refresh your dashboard
2. Check the Experts tab for any error messages
3. Verify token status on [Connections page](https://innova-trading.com/dashboard/connect)

---

### "Account Already Linked" Error

```
InnovaTrading EA: Error - Token already linked to different account
```

**Cause:** You're trying to use a token that was already used with a different MT4/MT5 account.

**Solution:**
1. Generate a new token for this account, OR
2. Disconnect the original account to free up the token

---

### Connection Drops Frequently

**Causes:**
1. Unstable internet connection
2. MetaTrader going to sleep/low priority
3. VPS resource constraints

**Solutions:**
1. Use a wired internet connection instead of WiFi
2. Disable MetaTrader power saving features
3. On VPS: Increase RAM allocation and disable Windows sleep

## API Errors

### 401 Unauthorized

```json
{
  "error": "unauthorized",
  "message": "Invalid API key"
}
```

**Causes:**
- Missing `Authorization` header
- Invalid API key format (should be `Bearer YOUR_KEY`)
- API key is less than 10 characters

**Solution:**
```bash
# Correct format
curl -H "Authorization: Bearer your_api_key_here" ...

# Wrong (missing Bearer)
curl -H "Authorization: your_api_key_here" ...
```

---

### 403 Forbidden - Symbol Not Allowed

```json
{
  "error": "forbidden",
  "message": "Symbol BTCUSDT not allowed for your API key",
  "allowed_symbols": ["EURUSD", "GBPUSD", "USDJPY", "XAUUSD"]
}
```

**Cause:** Your API key tier doesn't have access to this symbol.

**Solution:**
1. Use one of the `allowed_symbols` from the error message
2. Upgrade your plan to access more symbols
3. Call `GET /api/external/symbols` to see your available symbols

---

### 503 No Data Available

```json
{
  "error": "no_data",
  "message": "No bar data available. VPS/MT5 may be offline.",
  "symbol": "EURUSD",
  "timeframe": 60
}
```

**Cause:** The data source (VPS/MT5) is temporarily offline.

**Solutions:**
1. Wait a few minutes and retry
2. Check our [status page](https://status.innova-trading.com) for outages
3. Use cached data if you have it

<Warning>
  This error is temporary. Our VPS typically reconnects within 5-15 minutes.
</Warning>

---

### 422 Validation Error

```json
{
  "error": "validation_error",
  "message": "Point 0: type must be 'high' or 'low'",
  "point_index": 0
}
```

**Cause:** Invalid data in your request body.

**Common issues:**
- `type` must be exactly `"high"` or `"low"` (not `"HIGH"` or `"above"`)
- `time` must be a valid Unix timestamp
- `price` must be a number, not a string

---

### 404 Indicator Not Found

```json
{
  "error": "not_found",
  "message": "Indicator not found"
}
```

**Causes:**
1. Indicator hasn't been submitted yet
2. Indicator expired (data expires after 24 hours)
3. Wrong `indicator_id`, `symbol`, or `timeframe`

**Solution:**
```python
# Make sure all three match
GET /api/external/indicators/my_indicator?symbol=EURUSD&timeframe=60
```

## Rate Limiting

If you receive a `429 Too Many Requests` error:

```json
{
  "error": "rate_limit_exceeded",
  "message": "Too many requests. Limit: 100/hour",
  "retry_after": 3600
}
```

**Solutions:**
1. Implement exponential backoff
2. Cache responses where possible
3. Upgrade to a higher tier for more requests

## Debugging Tips

### 1. Check Your Request

```python
import requests

response = requests.get(
    "https://api.innova-trading.com/api/external/bars",
    params={"symbol": "EURUSD", "timeframe": 60, "limit": 10},
    headers={"Authorization": "Bearer YOUR_KEY"}
)

# Print full response for debugging
print(f"Status: {response.status_code}")
print(f"Headers: {response.headers}")
print(f"Body: {response.text}")
```

### 2. Verify Your API Key Works

```bash
# Test with symbols endpoint (simplest)
curl -v "https://api.innova-trading.com/api/external/symbols" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### 3. Check Indicator Data

```python
# List all your indicators
response = requests.get(
    "https://api.innova-trading.com/api/external/indicators",
    headers={"Authorization": "Bearer YOUR_KEY"}
)
print(response.json())
```
